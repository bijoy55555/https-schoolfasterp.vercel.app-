# স্কুল ERP — Firebase সেটআপ গাইড (সম্পূর্ণ বাংলা)

এই ফোল্ডারে আপনার স্কুল ERP সিস্টেমকে Firebase-এর সাথে যুক্ত করার সব ফাইল আছে।
নিচের ধাপগুলো একে একে অনুসরণ করুন।

---

## ধাপ ১: Firebase একাউন্ট ও প্রজেক্ট তৈরি করুন

1. ব্রাউজারে যান: **https://firebase.google.com**
2. উপরে ডান দিকে **"Go to console"** বাটনে ক্লিক করুন
3. আপনার যেকোনো **Gmail** দিয়ে লগইন করুন (নতুন Gmail না থাকলে প্রথমে একটা বানান)
4. **"Create a project" / "প্রজেক্ট তৈরি করুন"** বাটনে ক্লিক করুন
5. প্রজেক্টের নাম দিন, যেমন: `school-erp-bd` (ছোট হাতের অক্ষর, স্পেস ছাড়া)
6. **Google Analytics** অপশনটা এখন বন্ধ রাখলেও চলবে (পরে চালু করা যায়)
7. **"Create project"** ক্লিক করুন — কয়েক সেকেন্ড অপেক্ষা করুন

✅ এখন আপনার একটা Firebase প্রজেক্ট তৈরি হয়ে গেছে।

---

## ধাপ ২: Firestore ডাটাবেস চালু করুন

1. বাম দিকের মেনুতে **"Build" > "Firestore Database"** এ ক্লিক করুন
2. **"Create database"** বাটনে ক্লিক করুন
3. লোকেশন হিসেবে বেছে নিন: **`asia-south1` (Mumbai)** — বাংলাদেশের সবচেয়ে কাছের সার্ভার
4. **"Start in production mode"** বেছে নিন (সিকিউরিটি রুলস আমরা আগে থেকেই দিয়ে দিয়েছি)
5. **"Enable"** ক্লিক করুন

---

## ধাপ ৩: Authentication (লগইন সিস্টেম) চালু করুন

1. বাম মেনুতে **"Build" > "Authentication"** এ ক্লিক করুন
2. **"Get started"** ক্লিক করুন
3. **"Email/Password"** অপশনে ক্লিক করে **Enable** করে দিন, তারপর **Save**

---

## ধাপ ৪: ওয়েব অ্যাপ যুক্ত করে কনফিগারেশন কপি করুন

1. প্রজেক্ট হোমপেজে (বাম পাশে ⚙️ আইকনের পাশে) **"Project settings"** এ যান
2. নিচের দিকে **"Your apps"** সেকশনে **`</>`** (ওয়েব) আইকনে ক্লিক করুন
3. অ্যাপের নাম দিন: `school-erp-web`, তারপর **"Register app"**
4. একটা কোড ব্লক দেখাবে যেখানে `firebaseConfig` লেখা থাকবে — এটাই আপনার আসল চাবি

এরকম দেখাবে:
```
const firebaseConfig = {
  apiKey: "AIzaSyXXXXXXXXXXXXXXXXXXXXXX",
  authDomain: "school-erp-bd.firebaseapp.com",
  projectId: "school-erp-bd",
  storageBucket: "school-erp-bd.appspot.com",
  messagingSenderId: "123456789012",
  appId: "1:123456789012:web:abcdef123456"
};
```

5. এই পুরো জিনিসটা কপি করুন

## ধাপ ৫: কনফিগারেশন আপনার ফাইলে বসান

`public/index.html` ফাইলটা খুলুন (যেকোনো টেক্সট এডিটর দিয়ে, যেমন VS Code বা Notepad)।

ফাইলের শুরুর দিকে (`<head>` ট্যাগের ভেতরে) এই অংশটা খুঁজুন:

```javascript
const firebaseConfig = {
  apiKey: "আপনার_API_KEY",
  authDomain: "আপনার_প্রজেক্ট.firebaseapp.com",
  projectId: "আপনার_প্রজেক্ট_আইডি",
  storageBucket: "আপনার_প্রজেক্ট.appspot.com",
  messagingSenderId: "আপনার_SENDER_ID",
  appId: "আপনার_APP_ID"
};
```

এই পুরো অংশটা মুছে, ধাপ ৪ থেকে কপি করা আসল `firebaseConfig` বসিয়ে দিন। সেভ করুন।

## ধাপ ৬: কম্পিউটারে Firebase টুল ইনস্টল করুন

আপনার কম্পিউটারে আগে **Node.js** ইনস্টল থাকতে হবে (না থাকলে nodejs.org থেকে ডাউনলোড করুন)।

তারপর কমান্ড প্রম্পট / টার্মিনাল খুলে লিখুন:

```
npm install -g firebase-tools
```

ইনস্টল হয়ে গেলে লগইন করুন:

```
firebase login
```

(এটা ব্রাউজার খুলে আপনার Gmail দিয়ে লগইন করতে বলবে)

## ধাপ ৭: এই ফোল্ডারটা Firebase প্রজেক্টের সাথে যুক্ত করুন

1. `.firebaserc` ফাইলটা খুলুন
2. `"আপনার-প্রজেক্ট-আইডি-এখানে-বসান"` এর জায়গায় আপনার আসল Project ID বসান
   (Project ID পাবেন Firebase Console-এর Project Settings পেজে — উদাহরণ: `school-erp-bd`)

## ধাপ ৮: পেমেন্ট ও SMS এর গোপন চাবি (API Key) সেট করুন

**SSLCommerz একাউন্ট খোলার জন্য:**
- https://sslcommerz.com এ যান
- "Merchant Registration" করুন (ব্যবসার ট্রেড লাইসেন্স/স্কুলের রেজিস্ট্রেশন লাগতে পারে)
- প্রথমে **Sandbox (টেস্ট) একাউন্ট** দিয়ে শুরু করুন — এটা সাথে সাথে পাওয়া যায়, সম্পূর্ণ ফ্রি, টেস্টের জন্য
- Sandbox থেকে আপনি `Store ID` ও `Store Password` পাবেন

**SMS এর জন্য (BulkSMSBD বা যেকোনো লোকাল প্রোভাইডার):**
- https://bulksmsbd.net (বা আপনার পছন্দের প্রোভাইডার) এ একাউন্ট খুলুন
- তাদের থেকে `API Key` ও `Sender ID` সংগ্রহ করুন

এই চাবিগুলো টার্মিনালে এভাবে সেট করুন (ফোল্ডারের ভেতরে গিয়ে):

```
firebase functions:secrets:set SSLCOMMERZ_STORE_ID
firebase functions:secrets:set SSLCOMMERZ_STORE_PASSWORD
firebase functions:secrets:set SMS_API_KEY
firebase functions:secrets:set SMS_SENDER_ID
```

প্রতিটা কমান্ডের পর মান (value) লিখতে বলবে — কপি-পেস্ট করে Enter চাপুন।

## ধাপ ৯: সব কিছু আপলোড (ডিপ্লয়) করুন

## এখন যা কাজ করবে (ব্যাকএন্ড রেডি, কিন্তু ফ্রন্টএন্ড বাটনের সাথে এখনো জোড়া লাগেনি)

আমি `functions/index.js`-এ ৪টা ফাংশন রেডি করে দিয়েছি:

| ফাংশনের নাম | কাজ |
|---|---|
| `initiatePayment` | SSLCommerz দিয়ে পেমেন্ট লিংক তৈরি করে |
| `sendSms` / `sendBulkSms` | এসএমএস পাঠায় (একজনকে বা অনেককে) |
| `attendancePush` | বায়োমেট্রিক/RFID মেশিন থেকে হাজিরা ডেটা গ্রহণ করে |
| `dailyBackup` | প্রতিদিন রাত ২টায় স্বয়ংক্রিয় ব্যাকআপ নেয় |

⚠️ **পরের ধাপ:** আপনার `index.html`-এর বর্তমান পেমেন্ট/SMS বাটনগুলো এখনো শুধু UI সিমুলেশন
(সত্যিকারের ফাংশন কল করে না)। সেগুলোকে এই Cloud Functions-এর সাথে জোড়া লাগাতে হবে।
এটা পরবর্তী ধাপে করে দেওয়া যাবে — আগে আপনি Firebase একাউন্ট/প্রজেক্ট বানিয়ে
উপরের ধাপ ১-৯ শেষ করুন, তারপর জানান।

## 🏫 মাল্টি-স্কুল (একাধিক স্কুল) + সাবস্ক্রিপশন সিস্টেম — নতুন যোগ হয়েছে

এখন এই সিস্টেম দিয়ে **একই Firebase প্রজেক্টে একাধিক স্কুল** চালানো যাবে, প্রতিটার ডেটা
সম্পূর্ণ আলাদা (কেউ কারো ডেটা দেখতে পারবে না), এবং **মেয়াদ শেষ হলে অটো লগইন ব্লক** হয়ে যাবে।

### ✅ Cloud Function ছাড়াই — ফ্রি (Spark) প্ল্যানে চলে
এই সিস্টেমের মূল অংশ (স্কুল তৈরি, সাবস্ক্রিপশন/মেয়াদ নিয়ন্ত্রণ, মাল্টি-টেন্যান্ট
লগইন) এখন **সম্পূর্ণভাবে ব্রাউজার থেকেই কাজ করে** — কোনো Cloud Function ডিপ্লয়
বা Firebase Blaze (পেইড) প্ল্যান লাগে না। `firebase deploy` করারও দরকার নেই;
শুধু Firestore Rules একবার আপলোড করলেই হয় (নিচে ধাপ ক দেখুন)। SMS/পেমেন্ট/
বায়োমেট্রিক/অটো-ব্যাকআপ ফিচারগুলোর জন্য পরে Blaze প্ল্যানে আপগ্রেড করে
`functions/` ফোল্ডার ডিপ্লয় করা যাবে — সেটা ঐচ্ছিক এবং যেকোনো সময় করা যায়,
এখনকার ডেটা হারানো ছাড়াই।

### ধাপ ক — Firestore Rules আপলোড করুন
Firebase Console → Firestore Database → Rules ট্যাবে গিয়ে এই প্রজেক্টের
`firestore.rules` ফাইলের পুরো কনটেন্ট পেস্ট করে **Publish** করুন। (অথবা টার্মিনাল
থেকে `firebase deploy --only firestore:rules`।)

### ধাপ খ — নিজেকে সুপার-অ্যাডমিন বানান (একবারই)
1. Firebase Console → Authentication → Add user দিয়ে নিজের একটা ইমেইল/পাসওয়ার্ড একাউন্ট বানান
2. সেই ইউজারের **UID** কপি করুন (Authentication তালিকায় ইউজারের পাশে দেখা যাবে)
3. Firebase Console → Firestore Database → Start collection → নাম দিন `superAdmins`
4. ডকুমেন্ট আইডি হিসেবে সেই UID বসান, ভেতরে যেকোনো একটা ফিল্ড দিন (যেমন
   `isSuperAdmin: true`), তারপর Save
5. ব্রাউজারে যান: `https://আপনার-ডোমেইন/admin.html`, ধাপ ১-এ সেই ইমেইল/পাসওয়ার্ড
   দিয়ে লগইন করুন — "সুপার-অ্যাডমিন হিসেবে যাচাইকৃত" দেখালেই কাজ শেষ

### ধাপ গ — নতুন স্কুল (কাস্টমার) যোগ করুন
`admin.html` পেজেই "নতুন স্কুল যোগ করুন" ফর্ম পূরণ করুন — স্কুলের নাম, অ্যাডমিন
ইমেইল/পাসওয়ার্ড, প্যাকেজ, মেয়াদ (দিন)। বাটনে চাপলেই ব্রাউজার থেকে সরাসরি
(কোনো সার্ভার ছাড়াই) নতুন Firebase Auth একাউন্ট, `schools/{schoolId}` ডকুমেন্ট,
আর `userIndex/{uid}` লুকআপ ডকুমেন্ট তৈরি হয়ে যাবে। এরপর তারা সেই ইমেইল/পাসওয়ার্ড
দিয়ে মূল ERP (`index.html`)-এ লগইন করতে পারবে।

### কীভাবে কাজ করে
- প্রতিটা স্কুলের ডেটা `schools/{schoolId}/...` এর ভেতরে আলাদাভাবে থাকে
- কোন ইউজার কোন স্কুলের/কী রোল, তা `userIndex/{uid}` ডকুমেন্টে রাখা হয় (আগে এটা
  Firebase custom claim দিয়ে হতো, যেটার জন্য Cloud Function লাগত)
- লগইনের সময় ইমেইল/পাসওয়ার্ড ব্যবহার করলে (যেমন `admin@school.com`) আসল Firebase
  একাউন্ট দিয়ে লগইন হয়, `userIndex`-এ schoolId/role চেক হয়, এবং সাবস্ক্রিপশনের
  মেয়াদ যাচাই হয়
- আগের ডেমো ইউজারনেম (`admin`/`1234` ইত্যাদি, ইমেইল ফরম্যাট নয়) এখনো কাজ করবে —
  এটা শুধু টেস্ট/ডেমোর জন্য, কোনো Firestore-এ সেভ হয় না
- মেয়াদ বাড়ানো/সাসপেন্ড করা `admin.html`-এর স্কুল তালিকা থেকেই করা যায় (সরাসরি
  Firestore আপডেট, কোনো ফাংশন লাগে না)
- `functions/checkSubscriptions` (প্রতিদিন রাত ৩টায় অটো মেয়াদ-চেক + SMS রিমাইন্ডার)
  এখনো ঐচ্ছিক — Blaze প্ল্যানে আপগ্রেড করে ডিপ্লয় করলে বাড়তি সুবিধা হিসেবে চালু হবে

### ⚠️ এখনো যা বাকি (পরের ধাপ)
- এই মুহূর্তে প্রতিটা স্কুলের সব মডিউলের ডেটা (ছাত্র, ফি, হাজিরা, ফলাফল ইত্যাদি)
  একটা বড় Firestore ডকুমেন্টে বান্ডেল করে সিঙ্ক হয় (প্রতি ২০ সেকেন্ডে অটো-সেভ +
  সেটিংস/ছাত্র সেভ করলেও সিঙ্ক হয়)। ছোট-মাঝারি স্কুলের জন্য এটা ঠিক আছে, কিন্তু
  অনেক বড় স্কুল (হাজার হাজার ছাত্র/বছরের পর বছর হাজিরা ডেটা) হলে প্রতিটা মডিউল
  আলাদা Firestore কালেকশনে ভাগ করা ভালো — এটা পরের ধাপে করা যাবে।
- অ্যাডমিন প্যানেল (সব স্কুল একজায়গায় তালিকা, পেমেন্ট হিস্ট্রি, ম্যানুয়ালি
  মেয়াদ বাড়ানো) এখনো তৈরি হয়নি — `super-admin.html` আপাতত বেসিক টুল, চাইলে
  পূর্ণাঙ্গ ড্যাশবোর্ড পরে যোগ করা যাবে।

## 📊 গুগল শিট (Google Sheet) সিঙ্ক মডিউল — নতুন যোগ হয়েছে

প্রতিটা স্কুল এখন নিজের ছাত্র তালিকা, ফি ও হাজিরার ডেটা নিজস্ব একটা Google
Sheet-এ অটো-সিঙ্ক করতে পারবে — সব স্কুলের জন্য **একটাই** Google Cloud
Service Account ব্যবহার হয়, কিন্তু প্রতিটা স্কুলের ডেটা যায় শুধু তাদের
নিজেদের Sheet-এ। কোনো স্কুল অন্য স্কুলের Sheet দেখতে/লিখতে পারবে না, কারণ
কোন স্কুলের ডেটা কোন Sheet-এ যাবে সেটা সবসময় সার্ভারে লগইন করা ইউজারের
`userIndex/{uid}` থেকে বের করা schoolId দিয়ে ঠিক হয় — ক্লায়েন্ট থেকে schoolId
পাঠিয়ে অন্য স্কুলের Sheet-এ ঢোকার কোনো উপায় নেই।

### ধাপ ক — একটা Google Cloud Service Account বানান (একবারই)
1. https://console.cloud.google.com এ যান, একটা প্রজেক্ট বানান (বা আগের কোনোটা ব্যবহার করুন)
2. বাম মেনু থেকে **APIs & Services → Library** → "Google Sheets API" খুঁজে **Enable** করুন
3. **APIs & Services → Credentials → Create Credentials → Service Account** — যেকোনো নাম দিন, Create করুন
4. তৈরি হওয়া Service Account-এ ক্লিক করে **Keys → Add Key → Create new key → JSON** — একটা `.json` ফাইল ডাউনলোড হবে
5. এই JSON ফাইলের ভেতরে `client_email` ও `private_key` ফিল্ড দুটো পাবেন — এই দুটোই দরকার হবে

### ধাপ খ — Vercel-এ env variable সেট করুন
Vercel Dashboard → আপনার প্রজেক্ট → Settings → Environment Variables-এ এই দুটো যোগ করুন:

```
GOOGLE_SHEETS_CLIENT_EMAIL   = client_email এর মান (JSON থেকে)
GOOGLE_SHEETS_PRIVATE_KEY    = private_key এর মান (JSON থেকে, \n সহ পুরোটা কপি করুন)
```

সেভ করার পর প্রজেক্ট একবার **Redeploy** করুন। (এটা ঠিক `FIREBASE_PROJECT_ID` /
`FIREBASE_CLIENT_EMAIL` / `FIREBASE_PRIVATE_KEY` env variable তিনটার মতোই কাজ
করে, যেটা ইতিমধ্যে সেট করা আছে বলে ধরে নেওয়া হচ্ছে।)

### ধাপ গ — প্রতিটা স্কুল নিজে থেকে কানেক্ট করবে
স্কুল-অ্যাডমিন লগইন করে app.html-এর সাইডবার থেকে **"📊 গুগল শিট সিঙ্ক"**
(`sheets.html`) খুলবে, সেখানে দেখানো ইমেইলটাকে নিজের Google Sheet-এ Editor
হিসেবে শেয়ার করবে, Sheet-এর লিংক পেস্ট করে কানেক্ট করবে, তারপর "এখনই সিঙ্ক
করুন" চাপলেই Students/Fees/Attendance Logs — তিনটা ট্যাবে ডেটা চলে যাবে।
এটা কোনো Cloud Function ছাড়াই সরাসরি `api/sheets-config.js` ও
`api/sheets-sync.js` (Vercel serverless function) দিয়ে কাজ করে।

### আপনি (প্ল্যাটফর্ম মালিক) হিসেবে সব স্কুলের Sheet দেখতে চাইলে
যেহেতু Service Account-টা আপনার, এবং প্রতিটা স্কুল সেই একই ইমেইলকে Editor
বানায়, তাই আপনি চাইলে সরাসরি Google Drive-এ গিয়ে "Shared with me"-তে সব
স্কুলের Sheet একসাথে দেখতে পাবেন — আলাদা কোনো সেটআপ লাগে না।
