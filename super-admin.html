🛠️ সুপার-অ্যাডমিন প্যানেল

এখান থেকে নতুন স্কুল (টেন্যান্ট) যোগ করুন ও সাবস্ক্রিপশন দেখুন। শুধু আপনার (প্ল্যাটফর্ম মালিক) ব্যবহারের জন্য — এই লিংক কারো সাথে শেয়ার করবেন না।

ধাপ ১ — সুপার-অ্যাডমিন হিসেবে লগইন

এই একাউন্টটা সুপার-অ্যাডমিন হিসেবে যাচাই হয় Firestore-এর superAdmins/{আপনার UID} ডকুমেন্ট দিয়ে — এটা Firebase Console → Firestore Database থেকে একবার ম্যানুয়ালি বানাতে হয় (কোনো Cloud Function বা বিলিং লাগে না)। এখনো না বানিয়ে থাকলে, Authentication ট্যাব থেকে আপনার UID কপি করে, Firestore-এ superAdmins নামে একটা কালেকশন বানিয়ে, সেই UID-কে ডকুমেন্ট আইডি দিয়ে (ভেতরে যেকোনো একটা ফিল্ড, যেমন isSuperAdmin: true) সেভ করুন।

ধাপ ২ — নতুন স্কুল যোগ করুন

✅ এখন এই বাটনটা সম্পূর্ণভাবে ব্রাউজার থেকেই কাজ করে — কোনো Cloud Function বা Firebase Blaze (পেইড) প্ল্যান লাগে না। একটা লুকানো দ্বিতীয় Firebase সেশন দিয়ে নতুন অ্যাডমিন একাউন্ট বানানো হয়, তাই আপনার নিজের সুপার-অ্যাডমিন লগইন সেশন প্রভাবিত হবে না।

ধাপ ৩ — সব স্কুল (অ্যাডমিন প্যানেল)

স্কুলপ্যাকেজমূল্যমেয়াদ শেষঅবস্থাব্যবহারের অবস্থাকাজ
"তালিকা রিফ্রেশ করুন" বাটনে চাপুন
💡 ব্যবহারের অবস্থা কলামে দেখা যাবে কোনো স্কুল আসলেই ব্যবহার হচ্ছে কিনা (ছাত্র/হাজিরা ডেটা আছে কিনা)। অনেক সময় কেউ ফ্রি ট্রায়াল একাউন্ট খুলে আর ব্যবহার করে না — এমন স্কুল "⚠️ কোনো ডেটা ব্যবহার নেই" দেখাবে, যেগুলো নিশ্চিন্তে ডিলিট করা যায়।

🔔 ফ্রি প্ল্যান মনিটর — কবে পেইড প্ল্যানে যেতে হবে?

Firebase-এর Spark (ফ্রি) ও Vercel-এর Hobby (ফ্রি) প্ল্যানের একটা নির্দিষ্ট মাসিক/দৈনিক সীমা আছে। সঠিক রিয়েল-টাইম ব্যবহার (কত read/write/bandwidth হলো) শুধু Firebase Console ও Vercel Dashboard-এই দেখা যায় — এই ধরনের বিলিং ডেটা এই প্যানেল থেকে সরাসরি টেনে আনা সম্ভব না (তার জন্য আলাদা পেইড Cloud Monitoring API লাগবে)। তাই নিচে আপনার স্কুল/ছাত্র সংখ্যার ভিত্তিতে একটা আনুমানিক অবস্থা ও ফ্রি সীমার তালিকা দেওয়া হলো, এবং আসল সংখ্যা দেখার জন্য সরাসরি লিংক দেওয়া হলো — মাসে একবার লিংকগুলোতে ঢুকে দেখে নেওয়াই সবচেয়ে নিশ্চিত উপায়।

📊 দৈনিক Read/Write গ্রাফ কীভাবে দেখবেন:
  1. নিচের "Firestore Usage দেখুন" বাটনে চাপুন (সরাসরি সঠিক ট্যাবে নিয়ে যাবে)
  2. Firebase Console-এ আপনার সুপার-অ্যাডমিন গুগল একাউন্ট দিয়ে লগইন করুন
  3. Usage ট্যাবে Reads / Writes / Deletes — এই তিনটা আলাদা গ্রাফ দেখা যাবে (প্রতিদিনের হিসাব)
  4. গ্রাফের লাইন যদি ক্রমাগত উপরের ফ্রি-সীমার কাছাকাছি চলে যায় বা মাঝেমধ্যে ছুঁয়ে ফেলে (গ্রাফে লাল দাগ/সীমারেখা হিসেবে দেখানো থাকে), তখনই বুঝবেন Blaze প্ল্যানে (pay-as-you-go) সুইচ করার সময় হয়েছে
⚙️ আপনার প্রজেক্টের ১২টা Vercel Function — কোনটা বেশি চাপ ফেলে:
সবগুলো ফাংশনই একই কোটা (মাসিক ১০০ GB-Hrs) থেকে খরচ হয়, কিন্তু কতবার কল হচ্ছে তার উপর ভিত্তি করে কোনটা বেশি ঘন ঘন (ছাত্র/ডিভাইস সংখ্যার সাথে স্কেল করে) আর কোনটা কালেভদ্রে চলে সেটা আলাদা করে দেখানো হলো।
